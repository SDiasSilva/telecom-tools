QR Code Generator

About

This tool was raised for create QR codes based in a file with extension txt only. The file must have this format on all your lines: ICCID +White space+ LPA. Follow the example below.'

11111111111111111111 LPA:A2A2A2A2A2A2A2A2A2A2A2A2A2A2A2AA2A2A2A2A2A2A2A22A2AA2AA2A2A2A2A2A2.

Both them need be separated by one white space and, the iccid and lpa must be on the same line, being that the ICCID need be the first, and the LPA the last, i.e. the white space is in the middle of the two.



ErrorCorrectionlevelCntroller

The class "ErrorCorrectionlevelCntroller" is located in the namespace "Telecom_Tools.Controller" and depends on the "ErrorCorrectionLevel" class, which contains the properties LevelH, LevelL, LevelM, and LevelQ, as well as the methods SetLevelH, SetLevelL, SetLevelM, SetLevelQ, and SetErrorCorrectionLevel.

This class has the methods CriaSplitButtons() and ComboBox_SelectedIndexChanged().

CriaSplitButtons()

This method called "CriaSplitButtons" creates and configures a "ComboBox" to display error correction level options. The method is public and does not return a value. It receives a "comboBox" parameter, which is the ComboBox object to be configured. The method starts by creating a new string collection using the "StringCollection" class. Then, it adds the four error correction levels (H, L, M, and Q) to the collection, along with a prefix of "Level -" for each one. After that, it removes the "SelectedIndexChanged" event handler from the ComboBox and adds it again, setting the "ComboBox_SelectedIndexChanged" event handler. Next, it clears the items in the ComboBox and adds the string collection to the ComboBox, converting the strings to objects. Then, the method sets a pre-selected index of 2 (the M level), and if the pre-selected index is within the bounds of the ComboBox items, the method sets the selected index of the ComboBox to the pre-selected index. Otherwise, it sets the selected index to -1.

ComboBox_SelectedIndexChanged()

The "ComboBox_SelectedIndexChanged" method is an event handler for the "SelectedIndexChanged" event of the ComboBox. When the user selects an item in the ComboBox, the method is called and determines which item was selected. Then, it sets the corresponding error correction level in the "ErrorCorrectionLevel" class based on the selected item. The error correction level is determined by the values "H", "L", "M", and "Q" for the four items in the ComboBox.



ModuleWidthController

The class ModuleWidthController is located in the namespace "Telecom_tools.Controller" and depends on the ModuleWIdth class, which contains the properties moduleWith and moduleWidthLabelText that is used to set the text on the top of moduleWidth input on the view.

This class has the methods "CriaSplitButtons()" and "ComboBox_SelectedIndexChanged()".

CriaSplitButtons()

This method called "CriaSplitButtons" creates and configures a ComboBox to display options for module width. The method is public and does not return a value. It receives a parameter "comboBox", which is the ComboBox object to be configured. The method starts by creating a new string collection using the "StringCollection" class. Then, it uses a loop to add four strings representing module width values to the collection. The loop also removes and adds the "SelectedIndexChanged" event handler to the ComboBox each time an item is added. Next, it clears the items in the ComboBox and adds the collection of strings to the ComboBox, converting the strings to objects. The method sets a pre-selected index of 3 (representing a module width of 4), and if the pre-selected index is within the bounds of the ComboBox items, the method sets the ComboBox's selected index to the pre-selected index. Otherwise, it sets the selected index to -1.

ComboBox_SelectedIndexChanged()

The method "ComboBox_SelectedIndexChanged" is an event handler for the "SelectedIndexChanged" event of the ComboBox. When a user selects an item in the ComboBox, this method is called and sets the module width based on the selected index of the ComboBox. The module width is determined by adding 1 to the selected index, and the value is set using the "SetModuleWidth" method in the "ModuleWidth" class.

    This class has the methods "OpenFile()", "Generate()", "GenerateQR()", "GenerateQRCodeImages()", "GeneratePdf()", " CompressImages()", "DeleteAllPng()", "OpenFileExplorer()" and PutLogo().

    OpenFIle()

    The OpenFile method opens a dialog box to select a file. It displays the dialog box using the OpenFileDialog object, which is a Windows Forms component. It filters the files to display only text files and all files and waits for user selection. If the user selects a file, the file path is returned and stored in pathFile, and the selected file name is displayed in a text box that is passed as a parameter to the method. If the user cancels the file selection, the method returns null.

    Generate()

    The Generate method is the main method that coordinates the generation of PDF files and QR code images. It gets the name of the file to be processed using the GetFileName() method of the Model.File object. Then, it calls a series of auxiliary methods to generate the QR code images, PDF file, compress the QR code images into a ZIP file, delete the PNG files, and open the output folder in the file explorer. If an exception occurs, it is handled and an error message is displayed.

    GenerateQR()

    The GenerateQR method reads the text file selected in the OpenFile method and processes each line of the file. It separates the lines into two fields and stores each field in a list. Then, it checks if the number of elements in each list matches. If yes, it creates an output directory using the selected file name and generates the QR code images using the GenerateQRCodeImages method. If not, it throws an exception.

    GenerateQRCodeImages()

    The GenerateQRCodeImages method generates a QR code image for each input data pair (ICCID and LPAs) using the QRCodeGenerator library. It iterates over each data pair, creates a QR code using the data and provided configuration, and generates a PNG image from the QR code. If the "Include logo" option is selected, it adds the logo to the QR code image. The file name is the ICCID with the PNG extension.

    GeneratePDF()

    The GeneratePdf method generates a PDF file from the PNG files generated by the GenerateQRCodeImages method. It iterates over all PNG files in the output folder and adds them to the PDF file using the iTextSharp library. The PDF file is saved in the output folder with the same name as the input file, but with the PDF extension.

    CompressImages()

    The CompressImages method compresses PNG images into a ZIP file. It takes as parameters the name of the ZIP file to be generated, the directory where the PNG images were generated, and a CheckBox that indicates whether the user wants to compress the images into a ZIP file. The method checks if the CheckBox is checked and, if it is, iterates over all PNG images in the directory and adds them to the ZIP file using the ZipFile class from the System.IO.Compression library. The method uses the "Optimal" compression level and adds a 1-second pause after creating the ZIP file.

    DeleteAllPng()

    The DeleteAllPng method deletes all PNG files generated by the program. It takes as parameters the directory where the PNG files were generated and three CheckBoxes that indicate whether the user wants to generate a PDF file, a ZIP file, and/or keep the PNG files. The method checks if the generate a PDF or ZIP file CheckBox is checked and if the keep PNG files CheckBox is unchecked. If these conditions are met, the method iterates over all PNG files in the directory and deletes them using the System.IO.File.Delete method.

    OpenFileExplorer()

    The OpenFileExplorer method opens the Windows file explorer in a directory specified by the user. It displays a dialog box with a message asking if the user wants to open the directory. If the user clicks "Yes," the method uses the Process class from the System.Diagnostics library to open the specified directory.

    PutLogo()

    The PutLogo method adds a logo to a QR code image. It takes as parameters a CheckBox that indicates whether the user wants to add a logo and a QR code image. The method checks if the CheckBox is checked and, if it is, loads a logo image from the Resources folder of the program and resizes it proportionally to a size that corresponds to 1/5 of the width of the QR code image. Then, it positions the logo centered on the QR code image and returns it with the logo added. The method uses the Graphics class from the System.Drawing library to draw the logo on the QR code image.
